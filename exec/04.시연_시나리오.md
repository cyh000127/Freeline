# Freeline 시연 시나리오

## 1. 시연 목표

Freeline의 핵심 강점은 다음 4가지다.

1. 행사 관리자 관점의 행사/부스 운영 관리
2. 방문자 관점의 부스 조회 및 대기 등록
3. 부스 관리자 관점의 실시간 대기열 운영
4. 행사 종료 후 분석 리포트 생성

본 시나리오는 약 7~10분 분량 기준이다.

## 2. 시연 전 준비

### 2.1 권장 사전 데이터 준비

가장 안정적인 방식은 데모 스크립트를 미리 실행하는 것이다.

```bash
cd infra/loadtest
./run.sh demo --ready-pack
```

위 명령은 아래를 한 번에 수행한다.

- 시연용 행사 생성
- 부스/방문자 데이터 생성
- 부스 관리자 계정 생성
- 행동 로그/HDFS 적재
- 행사 종료(CLOSED)
- 리포트 생성 및 완료 대기

실행 후 생성되는 프로필 파일:

```text
infra/loadtest/output/demo-profile-event-<eventId>.md
```

이 파일에서 확인할 수 있는 정보:

- 행사 ID / 행사명
- 행사 관리자 로그인 정보
- 부스 관리자 로그인 ID
- 부스 관리자 고정 비밀번호 패턴(`DemoBooth!<boothId>`)
- 리포트 완료 여부

### 2.2 최소 점검 항목

- 관리자 웹: `/super` 접속 확인
- 부스 관리자 웹: `/booth` 접속 확인
- 모바일 앱 또는 에뮬레이터 실행 확인
- Backend `/actuator/health` 정상
- AI Vision `/health` 정상
- 리포트 기능 시연 시 Hive/HDFS 정상

## 3. 권장 시연 흐름

| 순서 | 화면/역할 | 시연 내용 | 예상 시간 |
| --- | --- | --- | --- |
| 1 | Super 관리자 | 행사 목록 진입, 행사 상세 확인, 부스/지도/행사 상태 소개 | 1분 |
| 2 | Super 관리자 | 행사 내 부스 데이터와 관리자 기능(상태 변경, 리포트 메뉴)을 보여줌 | 1분 |
| 3 | Mobile 방문자 | 행사 홈/지도/부스 상세 진입 후 특정 부스 대기 등록 | 1~2분 |
| 4 | Booth 관리자 | 대시보드에서 신규 대기 반영 확인, 다음 대기 호출 | 1분 |
| 5 | Mobile + Booth 관리자 | 호출 알림 확인 후 QR 또는 입장 처리, 대기 상태 변화 확인 | 1~2분 |
| 6 | Super 관리자 | 행사 종료 또는 종료 상태 확인 후 "분석 리포트 생성" 실행 | 1분 |
| 7 | Super 관리자 | 요약, 부스 성과, 시간대별 유입, 동선, 문제 지점 리포트 확인 | 1~2분 |

## 4. 상세 멘트 예시

### 4.1 관리자 파트

- "행사 관리자는 행사 정보, 부스 목록, 지도 이미지를 한 곳에서 관리할 수 있습니다."
- "행사 종료 후에는 관리자 화면에서 바로 분석 리포트를 생성할 수 있습니다."

### 4.2 방문자 파트

- "방문자는 모바일 앱에서 행사와 부스를 탐색하고, 원하는 부스에 대기를 등록할 수 있습니다."
- "지도와 부스 상세를 통해 어디에 어떤 부스가 있는지 즉시 확인할 수 있습니다."

### 4.3 부스 관리자 파트

- "부스 관리자는 웹 대시보드에서 대기열을 실시간으로 보고 호출/입장 처리를 할 수 있습니다."
- "호출 이후 상태 변화는 관리자 화면에 즉시 반영됩니다."

### 4.4 리포트 파트

- "행사 종료 후 DB dump와 행동 로그를 바탕으로 Hive 분석을 수행하고, 결과를 다시 서비스 DB로 적재해 관리자 리포트로 보여줍니다."

## 5. 안정성 위주 시연 방식

시간 제약이 크거나 네트워크 불안정이 우려되면 아래 순서를 권장한다.

1. `./run.sh demo --ready-pack` 으로 리포트까지 사전 완료
2. 관리자 화면에서 결과 리포트를 먼저 보여줌
3. 이후 모바일/부스 관리자 대기열 동작을 짧게 재현

이 방식은 라이브 리포트 생성 대기 시간을 줄일 수 있다.

## 6. 라이브 리포트 생성 시연 방식

리포트 생성 과정을 직접 보여주려면 아래 명령을 사용한다.

```bash
cd infra/loadtest
./run.sh demo --auto-close
```

이 경우:

- 행사 상태는 CLOSED까지 준비됨
- 시연자는 관리자 화면에서 "분석 리포트 생성"만 클릭하면 됨
- 완료까지 수 초에서 수십 초 정도 대기 가능

## 7. 비상 시 대체 시나리오

### 7.1 모바일 실기기 사용이 어려운 경우

- Expo 웹 또는 Android 에뮬레이터로 대체
- 또는 기존 생성된 방문자/대기 데이터를 기반으로 관리자 화면 중심 시연

### 7.2 QR 카메라 사용이 어려운 경우

- Booth 관리자 화면에서 호출과 입장 처리만 보여주고
- QR은 "도착 인증 수단"으로 설명만 보완

### 7.3 리포트 생성이 지연될 경우

- `demo-profile-event-<eventId>.md` 기준 완료된 행사로 전환
- 기존 완료 리포트 화면을 먼저 시연

## 8. 시연 체크리스트

- 행사 관리자 계정 로그인 정보 확보
- 부스 관리자 계정 로그인 정보 확보
- 대상 행사 ID 확인
- 모바일 앱 로그인 또는 방문자 진입 준비
- 리포트 완료 여부 확인
- 인터넷 연결 및 브라우저 탭 사전 준비

## 9. 추천 시연 구성

브라우저 탭은 최소 3개를 권장한다.

- 탭 1: 관리자(`/super`)
- 탭 2: 부스 관리자(`/booth`)
- 탭 3: 모바일 앱 또는 에뮬레이터

발표자는 관리자와 부스 관리자 화면을 동시에 열어 두면 실시간 반영을 보여주기 쉽다.
