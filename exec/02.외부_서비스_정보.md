# Freeline 외부 서비스 정보

## 1. 개요

Freeline은 단일 애플리케이션이 아니라 외부 SaaS와 별도 인프라 시스템을 함께 사용한다.  
아래 표는 포팅 시 반드시 확인해야 하는 연동 대상을 정리한 것이다.

## 2. 외부 SaaS / 공공 서비스

| 서비스 | 용도 | 연동 컴포넌트 | 필요 항목 | 필수 여부 | 장애 영향 |
| --- | --- | --- | --- | --- | --- |
| Cloudflare R2 | 이미지/파일 저장소 | Backend, User, Booth, Super | `CLOUDFLARE_ENDPOINT`, `CLOUDFLARE_BUCKET`, `CLOUDFLARE_ACCESS_KEY`, `CLOUDFLARE_SECRET_KEY`, `CLOUDFLARE_OUTER_PREFIX` | 선택 | 지도/부스/굿즈 이미지 업로드 및 삭제 실패 |
| Firebase Cloud Messaging | 모바일 푸시 알림 | Backend, Mobile | `FIREBASE_PROJECT_ID`, `FIREBASE_SERVICE_ACCOUNT_BASE64`, Android `google-services.json` | 선택 | 대기 호출/만료 푸시 발송 실패 |
| Gmail SMTP | 이메일 인증 메일 발송 | Backend | `MAIL_USERNAME`, `MAIL_PASSWORD`(앱 비밀번호 권장) | 선택 | 이메일 인증 기능 실패 |
| Daum Postcode | 주소 검색 팝업 | Super | 별도 키 없음(라이브러리 사용) | 선택 | 행사 주소 검색 UX 저하 |
| Expo EAS / Expo Updates | 모바일 APK 빌드 및 OTA | Mobile | Expo 계정, `EAS_PROJECT_ID`, `google-services.json` | 선택 | 클라우드 APK 빌드/업데이트 불가 |
| Let's Encrypt / Certbot | HTTPS 인증서 갱신 | Nginx | 도메인 DNS 제어 권한, `/etc/letsencrypt` 접근 | 운영 권장 | HTTPS 인증서 갱신 실패 |
| GitLab | 소스 저장소 및 webhook 트리거 | 전체 | GitLab 저장소 접근 권한 | 필수 | 소스 클론/CI 연동 불가 |

## 4. 운영 시크릿 관리 정보

원 운영 환경은 Infisical을 통해 시크릿을 주입한다.

### 4.1 사용 경로

- Backend: `/be`
- Frontend: `/fe`
- 운영 공통/인프라: `/prod`

### 4.2 CI에서 사용되는 주요 값

- `INFISICAL_URL`
- `INFISICAL_PROJECT_ID`
- `INFISICAL_CLIENT_ID`
- `INFISICAL_CLIENT_SECRET`

Infisical을 쓰지 않는 새 환경으로 포팅할 경우, 위 시크릿을 직접 `.env` 또는 Docker/CI 환경변수로 치환하면 된다.

### 4.3 Firebase 시크릿 등록 주의

- 백엔드용 Firebase 서비스 계정은 JSON 파일 경로를 직접 참조하지 않는다.
- 서비스 계정 JSON 원문을 Base64 문자열로 변환한 뒤 `FIREBASE_SERVICE_ACCOUNT_BASE64`에 저장해서 사용한다.
- Spring Boot는 시작 시 이 값을 메모리에서 Base64 디코딩하여 `GoogleCredentials`를 생성한다.
- 반면 `google-services.json`은 모바일 앱(Android) 빌드용 파일이므로 역할이 다르다.

## 5. 포팅 시 반드시 전달받아야 하는 계정/파일 목록

### 5.1 필수

- GitLab 저장소 접근 권한
- 서버 SSH 접근 권한
- Backend용 DB/Redis/RabbitMQ 접속 정보
- JWT/암호화 키
- Nginx 도메인 및 인증서 관리 권한

### 5.2 기능 전체 사용 시 추가 필요

- Cloudflare R2 버킷 및 API 키
- Firebase 프로젝트 및 서비스 계정 JSON 원본
- Android `google-services.json`
- Gmail SMTP 앱 비밀번호
- Expo 계정 및 EAS 프로젝트 권한
- Infisical 프로젝트 권한

## 6. 장애 영향 요약

| 항목 | 장애 시 영향 |
| --- | --- |
| Cloudflare R2 | 이미지 업로드/삭제 실패 |
| Firebase FCM | 푸시 알림 미발송 |
| Gmail SMTP | 이메일 인증 불가 |
| Hadoop/Hive | 리포트 생성 실패 |
| Infisical | 운영/CI 시크릿 주입 실패 |
| RabbitMQ | 실시간 대기열 이벤트 후처리 지연/실패 |
| Redis | 로그인/토큰/임시 상태 처리 문제 |
| PostgreSQL | 서비스 핵심 기능 전체 중단 |

## 7. 참고 메모

- Backend는 Firebase 미설정 시 NoOp sender로 내려가도록 되어 있어 빌드는 실패하지 않는다.
- Firebase를 실제로 사용하려면 서비스 계정 JSON 원본을 Base64로 변환해 `FIREBASE_SERVICE_ACCOUNT_BASE64`에 넣어야 한다.
- Cloudflare 관련 기능도 미설정 상태에서 앱 자체가 부팅 불가한 구조는 아니지만, 업로드 API는 실패한다.
- Daum Postcode는 관리자 주소 검색 편의 기능으로 핵심 API와는 분리되어 있다.
- 리포트 생성은 `DB dump -> Hive 분석 -> PostgreSQL 적재` 순서이므로 Hadoop/Hive가 없으면 관리자 리포트 메뉴는 정상 동작하지 않는다.
